# Changelog

All notable changes to the Railverse v2 project will be documented in this file. The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [2.0.0] - 2026-07-02

### Added
- **Repository Documentation**: Created the baseline project specifications and development requirements.
  - `README.md`: Overall introduction, tech stack details, performance benchmarks, and Git workflows.
  - `PROJECT_STRUCTURE.md`: Component layout definitions, folder hierarchies, and client/server BFF state flow.
  - `DEVELOPMENT_GUIDELINES.md`: TypeScript formatting rules, React Server Component guidelines, Tailwind CSS v4 styling rules, and Framer Motion animation presets.
  - `PRODUCT_REQUIREMENTS_DOCUMENT.md`: Central source of truth outlining the product vision, functional modules, and design goals.
  - `CONTRIBUTING.md`: Workflow configurations, pull request templates, and code review criteria.
  - `ROADMAP.md`: Detailed milestones for phases 1 through 6.
  - `SECURITY.md`: Vulnerability reporting process and security disclosure policy.
  - `CODE_OF_CONDUCT.md`: Contributor standards of behavior based on the Contributor Covenant.
- **Project Configuration**:
  - `.gitignore`: Configured Next.js, Node.js, and Windows workspace ignore file.
  - `LICENSE`: Added proprietary license placeholder for TravelCore Technologies.
  - `.github/PULL_REQUEST_TEMPLATE.md`: Structured PR details check.

### Changed
- Project version updated to `2.0.0` (Major rewrite starting from scratch).
